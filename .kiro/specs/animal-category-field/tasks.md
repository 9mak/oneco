# Implementation Plan: animal-category-field

## Tasks

- [x] 1. ドメインモデルにカテゴリフィールドを追加
- [x] 1.1 (P) RawAnimalData にカテゴリフィールドを追加
  - 収集した生データにカテゴリ情報を保持できるようにする
  - カテゴリは必須の文字列フィールドとして定義
  - 収集時点でアダプターが設定する値を受け取る
  - _Requirements: 1.1, 1.5_

- [x] 1.2 (P) AnimalData にカテゴリフィールドとバリデーションを追加
  - 正規化済みデータでカテゴリの 2 値制約（'adoption' / 'lost'）を保証
  - 無効な値が渡された場合はバリデーションエラーを発生させる
  - 既存の species, sex バリデーターと同じパターンで実装
  - _Requirements: 1.2, 1.3, 1.4, 1.5_

- [x] 1.3 (P) DataNormalizer でカテゴリをパススルー
  - 生データのカテゴリ値を正規化済みデータにそのまま渡す
  - 特別な変換処理は不要（アダプターで正しい値が設定される前提）
  - _Requirements: 1.2_

- [x] 2. データベース層にカテゴリ対応を追加
- [x] 2.1 Animal ORM モデルにカテゴリカラムを追加
  - カテゴリカラムを VARCHAR(20)、NOT NULL、デフォルト 'adoption' で定義
  - 単一カラムインデックスを作成してフィルタ性能を確保
  - 複合検索インデックスにカテゴリを含めて最適化
  - _Requirements: 2.1, 2.2, 2.5_

- [x] 2.2 Alembic マイグレーションを作成
  - 既存テーブルにカテゴリカラムを追加するマイグレーションを作成
  - 既存データにはデフォルト値 'adoption' を自動設定
  - ロールバック時にカラムを安全に削除できるようにする
  - _Requirements: 2.3, 2.4, 5.1, 5.4_

- [x] 2.3 AnimalRepository にカテゴリフィルタを追加
  - 一覧取得メソッドにカテゴリパラメータを追加
  - カテゴリ指定時は該当カテゴリのみをフィルタ、省略時は全件を返却
  - Pydantic と ORM 間の変換でカテゴリフィールドをマッピング
  - _Requirements: 3.3, 3.4, 3.5_

- [x] 3. アダプター層にカテゴリ判定を実装
- [x] 3.1 MunicipalityAdapter インターフェースを拡張
  - 一覧取得メソッドの返却型を URL とカテゴリのペアリストに変更
  - 詳細抽出メソッドにカテゴリ引数を追加
  - 将来の他自治体アダプターでも同じパターンを使用
  - _Requirements: 4.3, 4.4_

- [x] 3.2 KochiAdapter にカテゴリ判定ロジックを実装
  - 譲渡情報ページからの収集時に 'adoption' を設定
  - 迷子情報ページからの収集時に 'lost' を設定
  - 判定できない場合はデフォルト 'adoption' を設定し警告ログを出力
  - 抽出した生データにカテゴリを含めて返却
  - _Requirements: 4.1, 4.2, 4.5_

- [x] 3.3 CollectorService のデータフローを更新
  - アダプターから URL とカテゴリのペアを受け取る
  - 各詳細ページ抽出時にカテゴリを渡す
  - 既存のエラーハンドリングとリトライロジックを維持
  - _Requirements: 4.1, 4.2_

- [x] 4. API 層にカテゴリ対応を追加
- [x] 4.1 (P) AnimalPublic スキーマにカテゴリフィールドを追加
  - API レスポンスにカテゴリ情報を含める
  - 既存クライアントは新フィールドを無視可能（後方互換）
  - _Requirements: 3.1, 5.3_

- [x] 4.2 (P) GET /animals にカテゴリクエリパラメータを追加
  - カテゴリパラメータをオプショナルで受け取る
  - 有効値（'adoption', 'lost'）でフィルタを適用
  - 無効値の場合は HTTP 400 エラーを返却
  - カテゴリ省略時は全カテゴリのデータを返却
  - _Requirements: 3.2, 3.3, 3.4, 3.5, 3.6, 5.2_

- [x] 5. テストと検証
- [x] 5.1 ドメインモデルのユニットテストを更新
  - RawAnimalData のカテゴリフィールド存在を確認
  - AnimalData の有効カテゴリ値の受け入れを確認
  - AnimalData の無効カテゴリ値での ValidationError を確認
  - DataNormalizer のカテゴリパススルーを確認
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 5.2 データベース層のテストを更新
  - マイグレーションの upgrade/downgrade 動作を確認
  - Repository のカテゴリフィルタ動作を確認
  - ORM と Pydantic 間のカテゴリマッピングを確認
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 3.3, 3.4, 3.5_

- [x] 5.3 アダプター層のテストを更新
  - fetch_animal_list が URL とカテゴリのペアを返却することを確認
  - 譲渡情報ページからの収集で 'adoption' が設定されることを確認
  - 迷子情報ページからの収集で 'lost' が設定されることを確認
  - extract_animal_details がカテゴリ付き RawAnimalData を返却することを確認
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 5.4 API 層のテストを更新
  - category=adoption でのフィルタ動作を確認
  - category=lost でのフィルタ動作を確認
  - category 省略時の全件返却を確認
  - 無効カテゴリでの 400 エラーを確認
  - レスポンスにカテゴリフィールドが含まれることを確認
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6_

- [x] 5.5 エンドツーエンドの統合テスト
  - アダプターからの収集 → 正規化 → DB 保存 → API 返却の全フローを検証
  - カテゴリ情報が各レイヤーで正しく伝播することを確認
  - マイグレーション適用後の既存データにデフォルトカテゴリが設定されていることを確認
  - source_url の一意性制約が維持されていることを確認
  - _Requirements: 1.1, 1.2, 2.4, 4.1, 4.2, 5.1, 5.3, 5.5_
