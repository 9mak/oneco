## 技術方針

### データ抽出アーキテクチャ移行
```
[現在] サイト固有アダプター（ルールベース）
  └─ 高知県: kochi_adapter.py（450行、特別ルール4つ）
  └─ 課題: 1サイトあたり数時間〜1日、保守コスト大

[Phase 1以降] AI抽出エンジン（LLMベース）
  └─ 汎用エンジン + YAML設定
  └─ 新規サイト追加: 設定数行のみ
  └─ サイト構造変更にも頑健
```

### サイト設定例（想定）
```yaml
sites:
  - name: "徳島県動物愛護管理センター"
    prefecture: "徳島県"
    list_url: "https://douai-tokushima.com/"
    list_link_pattern: "a[href*='detail']"  # 最低限のヒント
    extraction: "llm"  # AI抽出
```
